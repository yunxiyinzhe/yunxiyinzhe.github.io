Eclipse RCP 移除/隐藏(remove or hide)menu，toolbar等UI元素的方式

作者：**Dylan Gao**

出处：**http://dylangao.com/index.php/2013/09/23/73/**

声明：**本文采用以下协议进行授权：自由转载-非商用-非衍生-保持署名|Creative Commons BY-NC-ND 3.0转载请注明作者及出处。**

前言

在做 Eclipse RCP 开发的时候，会有一些冗余的UI元素显示在界面上，这些元素或是Eclipse本身自带的，或是其他依赖插件添加进去的。为了保持界面的整洁，就需要去除或隐藏掉这些界面元素。

刚刚接到这个任务的时候，本身对Eclipse开发了解不深的我几乎无从下手，最后全靠google，stackoverflow和eclipse社区的帮助，才在黑暗中慢慢摸索出一条路，额，其实是好多条路。

遇到这种比较”非主流“的技术问题，google很重要，英文很重要，查阅source code很重要。

本文几乎包含了网上能找到的，以及自己摸索出的各种“移除/隐藏” UI元素的方法。

学艺不深，欢迎指点。

主Menu的移除

1.实现自己的ActionBarAdvisor类

在RCP理的ApplicationWorkbenchWindowAdvisor（继承自WorkbenchWindowAdvisor）类中，有一个函数：

public ActionBarAdvisor createActionBarAdvisor(IActionBarConfigurer configurer) {
    return new ApplicationActionBarAdvisor(configurer);
}

这里面的ApplicationActionBarAdvisor就是自己要实现的继承自ActionBarAdvisor的类，这个类里面，包含了几乎所有主Menu菜单的创建过程，所以如果在这里删除某个menu，就是从根本上删除某个menu（后面有动态添删除menu的方法）。比如File menu下面的save menu item的创建过程就可以简化为：

private IWorkbenchAction saveAction;

private MenuManager createFileMenu() {
    MenuManager menu = new MenuManager(IDEWorkbenchMessages.Workbench_file, IWorkbenchActionConstants.M_FILE);
    menu.add(saveAction);

}
protected void makeActions(final IWorkbenchWindow window) {
    saveAction = ActionFactory.SAVE.create(window);
    register(saveAction);
}

2.获取MenuManager动态管理

获取主menu的MenuManager，然后遍历所有取得的menus，可以实现动态管理menu，实例代码如下所示：

IWorkbenchWindow window = Workbench.getInstance()
        .getActiveWorkbenchWindow();
MenuManager menuManager = ((WorkbenchWindow) window).getMenuManager();

IContributionItem[] menuItems = menuManager.getItems();
for (IContributionItem menuItem : menuItems) {
    //manage your menu here, and call setVisible(false) to hide menu
}

判断menuItem的具体类型，以实现更加灵活的menu管理

主Toolbar的移除

1.实现自己的ActionBarAdvisor类

和menu类似，在自己实现的ActionBarAdvisor类里面也包含了toolbar的创建过程，所以我们当然可以在这里实现对toolbar的删除。toolbar是在函数fillCoolBar中创建的

protected void fillCoolBar(ICoolBarManager coolBar) {

    IActionBarConfigurer2 actionBarConfigurer = (IActionBarConfigurer2) getActionBarConfigurer();
    { // Set up the context Menu
        coolbarPopupMenuManager = new MenuManager();
        coolbarPopupMenuManager.add(new ActionContributionItem(lockToolBarAction));
        coolbarPopupMenuManager.add(new ActionContributionItem(editActionSetAction));
        coolBar.setContextMenuManager(coolbarPopupMenuManager);
        IMenuService menuService = (IMenuService) window.getService(IMenuService.class);
        menuService.populateContributionManager(coolbarPopupMenuManager, "popup:windowCoolbarContextMen"); //$NON-NLS-1$
    }
    coolBar.add(new GroupMarker(IIDEActionConstants.GROUP_FILE));
    { // File Group
        IToolBarManager fileToolBar = actionBarConfigurer.createToolBarManager();
        fileToolBar.add(new Separator(IWorkbenchActionConstants.NEW_GROUP));
        fileToolBar.add(newWizardDropDownAction);
        fileToolBar.add(new GroupMarker(IWorkbenchActionConstants.NEW_EXT));
        fileToolBar.add(new GroupMarker(
                IWorkbenchActionConstants.SAVE_GROUP));
        fileToolBar.add(saveAction);
        fileToolBar.add(saveAllAction);
        fileToolBar
                .add(new GroupMarker(IWorkbenchActionConstants.SAVE_EXT));
        fileToolBar.add(getPrintItem());
        fileToolBar
                .add(new GroupMarker(IWorkbenchActionConstants.PRINT_EXT));

        fileToolBar
                .add(new Separator(IWorkbenchActionConstants.BUILD_GROUP));
        fileToolBar
                .add(new GroupMarker(IWorkbenchActionConstants.BUILD_EXT));
        fileToolBar.add(new Separator(
                IWorkbenchActionConstants.MB_ADDITIONS));

        // Add to the cool bar manager
        coolBar.add(actionBarConfigurer.createToolBarContributionItem(fileToolBar,
                IWorkbenchActionConstants.TOOLBAR_FILE));
    }
}

2.获取ActionSet管理toolbar

toolbar上的操作按钮对应actionset，所以获取了ActionSet就能实现对toolbar按钮的删除，实例代码如下:

WorkbenchPage worbenchPage = (WorkbenchPage) PlatformUI.getWorkbench()
        .getActiveWorkbenchWindow().getActivePage();
IActionSetDescriptor[] actionSets = worbenchPage.getActionSets();
for (IActionSetDescriptor action : actionSets) {
    //manage your toolbar here, and call worbenchPage.hideActionSet(actionID) to hide toolbar
}

利用扩展点org.eclipse.ui.activities隐藏UI元素

怕自己讲不清出，先引用一个blog：http://blog.vogella.com/2009/07/13/eclipse-activities/

然后是官方的help：http://help.eclipse.org/indigo/index.jsp?topic=%2Forg.eclipse.platform.doc.isv%2Freference%2Fextension-points%2Forg_eclipse_ui_activities.html

官方解释如下：

The org.eclipse.ui.activities extension point is used to declare activities and associated elements. Activities are used by the platform to filter certain plugin contributions from the users view until such a time that they express interest in them. This allows Eclipse to grow dynamically based on the usage pattern of a user.

看了解释，说明这个扩展点是用来过滤掉某些插件的contributions的，我们一般用它来控制UI元素的显示和隐藏。其使用很简单，如下：

这个插件的使用难点在于找准你要过滤的UI元素的ID，然后填到pattern里面，pattern可以填入正则表达式，例如你想要过滤所有PreferencePage相关，只要：

pattern=”.org.eclipse.ui.preferencePages.“

右键菜单的移除(Navigator,Edit,View的右键菜单)

1.Navigator右键菜单的移除

在Navigator的实现类（继承了IMenuListener接口）里面，可以重写父类的menuAboutToShow函数实现对右键菜单的管理。

menuAboutToShow函数传入的IMenuManager manager正是我们需要的对象，实例代码如下：

public void menuAboutToShow(IMenuManager menu) {
    super.menuAboutToShow(menu);

    IContributionItem[] items = menu.getItems();
    for (int i = 0; i < items.length; i++) {
                 //manage your menu, and call setVisible(false) to hide menu
    }
}

2.Editor右键菜单的移除

和Navigator类似，Editor里面也有菜单打开的监听方法editorContextMenuAboutToShow，也会传入IMenuManager menu，因此处理方法也类似

protected void editorContextMenuAboutToShow(IMenuManager menu) {
    super.editorContextMenuAboutToShow(menu);
    IContributionItem[] items = menuManager.getItems();
    for (int i = 0; i < items.length; i++){
     //manage your menu, and call setVisible(false) to hide menu
    }
}

3.View右键菜单的移除

这里一Debug Perspective下面的Debug View为例，其他View可能也可以用相同的思路去除某些menu。

查看Debug View的源代码，继承自AbstractDebugView，发现有以下代码：

protected void createContextMenu(Control menuControl) {
    MenuManager menuMgr= new MenuManager("#PopUp"); //$NON-NLS-1$
    menuMgr.setRemoveAllWhenShown(true);
    menuMgr.addMenuListener(new IMenuListener() {
        public void menuAboutToShow(IMenuManager mgr) {
            fillContextMenu(mgr);
        }
    });
    Menu menu= menuMgr.createContextMenu(menuControl);
    menuControl.setMenu(menu);

    // register the context menu such that other plug-ins may contribute to it
    if (getSite() != null) {
        getSite().registerContextMenu(menuMgr, getViewer());
    }
    addContextMenuManager(menuMgr);
}

显然这里就是创建右键菜单的地方，menuMgr注册了IMenuListener，后者实现了menuAboutToShow，所以这里就是突破口。

在我们的代码里面，在需要去除Debug View的地方，我们也可以在右键菜单的MenuManager里面注册我们自己的IMenuListener，实现对菜单的管理，代码如下：

LaunchView launchView = (LaunchView)page.findView("org.eclipse.debug.ui.DebugView");
if(launchView != null) {
    List contextMenuMgr = launchView.getContextMenuManagers();
    for(MenuManager mgr : contextMenuMgr) {
        mgr.addMenuListener(new IMenuListener() {

            @Override
            public void menuAboutToShow(IMenuManager manager) {
                // TODO Auto-generated method stub
                manager.remove("org.eclipse.debug.ui.debugview.toolbar.disconnect");
                manager.remove("org.eclipse.debug.ui.debugview.toolbar.dropToFrame");
                manager.remove("org.eclipse.debug.ui.actions.ToggleStepFilters");
            }
        });
    }
}

View上toolbar按钮的移除

view上toolbar按钮的管理比较简单，只需要获取相应View的ToolBarManager，然后就可以为所欲为了，以VariableView作为例子，代码实例如下

VariablesView view = (VariablesView) page.findView("org.eclipse.debug.ui.VariableView");
if (view != null) {
    IToolBarManager tbm = view.getViewSite().getActionBars().getToolBarManager();
    tbm.remove("org.eclipse.debug.ui.ShowTypesAction");
    tbm.remove("org.eclipse.debug.ui.ToggleObjectBrowsersAction");
}
