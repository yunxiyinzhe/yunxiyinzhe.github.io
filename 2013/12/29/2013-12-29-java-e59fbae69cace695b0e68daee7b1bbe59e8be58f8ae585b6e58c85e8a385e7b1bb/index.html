Java 基本数据类型及其包装类

技术
  • Java学习

1.基本数据类型

Java基本数据类型(Primitive Types)可以分为数值类型(Numeric Types)、 布尔类型 (Boolean Type)和 returnAddress 类型三类。其中数值类型又分为整型类型和浮点类型.

1.1整数类型

整数类型包括以下几种类型:

  • byte 类型：值为 8 位有符号二进制补码整数，默认值为零。取值范围是从-128 至 127(-2的7次方至 2的7次方7-1)，包括-128 和 127。
  • short 类型：值为 16 位有符号二进制补码整数，默认值为零。取值范围是从−32768 至 32767(-2的7次方15至 2的7次方15-1) ，包括−32768 和32767。
  • int 类型：值为 32 位有符号二进制补码整数，默认值为零。取值范围是从−2147483648 至 2147483647(-2的7次方31至 2的7次方31-1)，包括−2147483648 和 2147483647。
  • long 类型：值为 64 位有符号二进制补码整数，默认值为零。取值范围是从−9223372036854775808 至 9223372036854775807(-2的7次方63至 2的7次方63-1) ，包括−9223372036854775808 和 9223372036854775807。
  • char 类型：值为使用 16 位无符号整数表示的、指向基本多文本平面（Basic Multilingual Plane， BMP）的 Unicode 值，以 UTF-16 编码，默认值为 Unicode的 null 值(‘\u0000’)。取值范围是从 0 至 65535，包括 0 和 65535。

有以下几点需要注意:

  • 如果将超过变量类型取值范围的数值赋给变量,会产生编译错误,即编译器会检查变量的取值范围
byte bb = 200;//compile error
short ss = 60000;//compile error
int ii = 4000000000;//compile error
long ll = 10000000000000000000L;//compile error
char cc = 70000;//compile error
  • 如果一个整数超过了int的取值范围,则这个整数常量必须加上L或l才能赋值给long类型变量
long l1 = 2000000000;//compile OK
long l2 = 900000000000000000;//compile error
long l3 = 900000000000000000L;//compile OK
  • Java中整数常量有三种表示方式:10进制,8进制(以0开头),16进制(以0x或0X开头)
int octalValue = 013;//十进制1*8+3 = 11
int hexValue1 = 0x13;//十进制1*16+3 = 19
int hexValue2 = 0Xab;//十进制10*16+11 = 171
System.out.println(octalValue);
System.out.println(hexValue1);
System.out.println(hexValue2);

1.2浮点类型

  • float 类型：值为单精度浮点数集合中的元素，或者(如果虚拟机支持的话)是单精度扩展指数（Float-Extended-Exponent）集合中的元素。默认值为正数零。
  • double 类型：取值范围是双精度浮点数集合中的元素，或者(如果虚拟机支持的话)是双精度扩展指数(Double-Extended-Exponent）集合中的元素。默认值为正数零。

Java的默认浮点类型是double,如果想要把一个浮点常量当做float,则需要在常量后面加上F或f

float f1 = 1.0;//compile error
float f2 = 1.0F;
double d = 1.0;

Java提供了三个特殊的浮点数值:正无穷大(POSITIVE_INFINITY)、负无穷大(NEGATIVE_INFINITY)和非数(NaN). 正数除以0会得到正无穷大，负数除以0会得到负无穷大，0.0除以0.0或对负数开方会得到非数。 注意Java里面只有浮点数除以0才会得到正无穷大或负无穷大，整数除以0则会得到一个异常

double dpi = 1.0/0;
System.out.println(dpi);
double dni = -1.0/0;
System.out.println(dni);
int ipi = 1/0;
System.out.println(ipi);

输出为

Infinity
-Infinity
Exception in thread "main" java.lang.ArithmeticException: / by zero
    at Test.main(Test.java:83)

正无穷大用Double或Float的POSITIVE_INFINITY表示，负无穷大用Double或Float的NEGATIVE_INFINITY表示，非数用Double或Float的NaN表示。 所有正无穷大相等，所有负无穷大相等，非数都不相等

System.out.println(Float.NEGATIVE_INFINITY == Double.NEGATIVE_INFINITY);//true
System.out.println(Float.POSITIVE_INFINITY == Double.POSITIVE_INFINITY);//true
System.out.println(Float.NaN == Float.NaN);//false

1.3布尔类型

  • boolean 类型：取值范围为布尔值 true 和 false，默认值为 false

boolean类型的取值只能是true或 false，不能用0或非0来表示，其他类型不能转换为boolean类型。如果boolean类型和字符串连接，则boolean自动转换为字符串

System.out.println(true+"");//true

1.4基本成员的默认值

当类的成员变量是基本数据类型的时候,即使不对成员变量进行初始化,Java也会将其自动初始化为一个默认值

boolean—->false char—->’\u0000’ byte—->(byte)0 short—->(short)0

int—->0 long—->0L float—->0.0f double—->(double)0.0d

但是对于局部变量,则不会有默认初始化.

2.基本数据类型的类型转换

Java基本数据类型之间的类型转换可以分为：自动类型转换和强制类型转换

2.1自动类型转换

如果系统支持把某个基本数据类型的值直接赋值给另一个基本数据类型，则这种方式就成为自动类型转换。当把一个表数范围较小的数值或变量赋给另一个表数范围较大的变量的时候，系统可以进行自动类型转换。Java支持自动类型转换的类型如图

byte b = 50;
short s = b;
int i = s;
long l = i;
float f = l;
double d = f;
System.out.println(b);//50
System.out.println(s);//50
System.out.println(i);//50
System.out.println(l);//50
System.out.println(f);//50.0
System.out.println(d);//50.0

char c = b;//compile error
int ii = 1.0;//compile error
double ff = 1.0;
long ll = ff;//compile error

把任何基本类型的值和字符串进行连接运算时，基本类型的值将自动转换为字符串类型。如下代码所示：

//String s1 = 5;//compile error
String s1 = 5 + "";
String s2 = 5.0 + "";
String s3 = true + "";
System.out.println(s1);//5
System.out.println(s2);//5.0
System.out.println(s3);//true

2.2强制类型转换

如果把上图箭头右边的类型转换为左边的类型，则必须进行强制类型转换。注意，强制类型转换可能产生数据溢出。

int i1 = 300;
byte b = (byte)i1;
float f = 5.9f;
int i2 = (int)f;
System.out.println(b);//44 数据溢出
System.out.println(i2);//5 浮点型强制转换为整型，截断小数点

当把一个浮点数强制转换为整数时，Java会直接截断浮点数的小数部分。如果想得到四舍五入的结果，则需要调用java.lang.Math中的round()方法。当把一个超过某个类型表数范围的数值或变量赋给这个类型变量的时候，则可能产生数据溢出。

2.3表达式类型的自动提升

当一个算术表达式包含多种数据类型的时候，整个算术表达式的数据类型将发生自动提升。规则如下：

  • byte，short，char类型将被提升到int类型
  • 整个算术表达式的数据类型自动提升到与表达式中最高等级操作数同样的类型

下面是一个典型的错误

short s = 5;
s = s - 2;//compile error

s-2表达式提升到int类型，不能直接将int复制给short。

表达式类型将严格保持与表达式中等级最高的操作数类型相同。两个整数相除，得到的结果的小数部分会被截断。

int a = 15;
int b = 8;
System.out.println(a/b);// 15/8 = 1.875,截断为1

3.基本数据类型的包装类

Java里面8种基本数据类型都有其对应的包装类：

基本数据类型 包装类
byte Byte
char Character
short Short
float Float
int Integer
double Double
long Long
boolean Boolean

3.1包装类对象的构造

可以通过调用包装类的构造器将基本数据类型包装成包装类实例。而且除了Character外，其他包装类都可以传入一个字符串来构造包装类对象。当用字符串创建Boolean对象时，字符串大小写不敏感。当传入包装类的字符串无法转换为相应基本类型变量时，则抛出java.lang.NumberFormatException异常。

boolean b = false;
Boolean B1 = new Boolean(b);    
Boolean B2 = new Boolean("TRuE");//OK
Boolean B3 = new Boolean("fAlse");//OK
Integer I = new Integer(5);
Float F1 = new Float(5.5);//Float(double value) 可以传入double值
Float F2 = new Float(5.5f);
ouble D = new Double("5.6");
//Character C = new Character("?")//compile error
Long L = new Long("sdsds");//Exception in thread "main" java.lang.NumberFormatException

3.2自动装箱与自动拆箱

JDK1.5后提供自动装箱与自动拆箱功能。自动装箱就是把一个基本类型变量直接赋值给包装类变量(或Object)，自动拆箱是把一个包装类变量直接赋给一个基本类型变量。

Integer I = 5;//AutoBoxing
Object O = true;//AutoBoxing
if(O instanceof Boolean) {
    Boolean B = (Boolean)O;//AutoBoxing
    boolean b = (Boolean)O;//AutoUnBoxing
}
int i = I;//AutoUnBoxing

3.3基本类型变量与字符串

包装类还可以实现基本类型变量与字符串之间的转换，有两种方式：

  • 包装类的parseXXX(String s)静态方法(除了Character类外)
  • 包装类的构造器方法

String类还提供多个重载方法valueOf()，用于将基本类型变量转换为字符串，下面是实例程序

Integer I = Integer.parseInt("50");
Double D = new Double("5.0");
String fstr = String.valueOf(5.3f);

3.4Integer类的特殊情况

先看下面的程序

Integer ia = new Integer(20);
Integer ib = new Integer(200);
Integer i1 = 20;
Integer i2 = 20;
Integer I1 = 200;
Integer I2 = 200;
System.out.println(i1 == ia);//false
System.out.println(i1 == i2);//true
System.out.println(I1 == ib);//false
System.out.println(I1 == I2);//false

查看Integer源代码可以发现，系统把-127～127之间的整数自动装箱成Integer实例，并放入了一个名为cache的数组中缓存起来。以后如果再有一个-127～127之间的整数自动装箱成Integer实例，则直接指向cache数组中的元素。因此对上面的程序，调用new构造器产生的实例在堆上，每次都会产生新的对象，-127～127之间的整数自动装箱成Integer实例，无论调用多少次，都指向同一个对象。继续看下面的实例

Integer m = new Integer(5);  
Integer n = new Integer(5);  
System.out.println(m==n);//false
m = m-1;  
n = n-1;  
System.out.println(m==n);//true

m,n因为都是new出来的对象，内存地址不一样，所以第一次m==n比较的reference不一样。但是，m=m-1首先进行了自动拆箱 m.intValue，相减后再进行装箱动作：m=Integer.valueOf(m.intValue-1)，而m和n都在 -128–127的范围，所以自动装箱后，根据上文所述，都是同一个object的引用。因此第二次输出true。
