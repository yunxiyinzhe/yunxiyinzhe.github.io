前言

在做 Eclipse RCP 开发的时候，会有一些冗余的UI元素显示在界面上，这些元素或是Eclipse本身自带的，或是其他依赖插件添加进去的。为了保持界面的整洁，就需要去除或隐藏掉这些界面元素。

刚刚接到这个任务的时候，本身对Eclipse开发了解不深的我几乎无从下手，最后全靠google，stackoverflow和eclipse社区的帮助，才在黑暗中慢慢摸索出一条路，额，其实是好多条路。

遇到这种比较”非主流“的技术问题，google很重要，英文很重要，查阅source code很重要。

本文几乎包含了网上能找到的，以及自己摸索出的各种“移除/隐藏” UI元素的方法。

学艺不深，欢迎指点。

Java程序员应该了解的10个面向对象设计原则-CSDN.NET

原文：http://www.csdn.net/article/2012-03-14/313089


摘要：Java编程最基本的原则就是要追求高内聚和低耦合的解决方案和代码模块设计。查看Apache和Sun的开放源代码能帮助你发现其他Java设计原则在这些代码中的实际运用。

面向对象设计原则是OOPS（Object-Oriented Programming System，面向对象的程序设计系统）编程的核心，但大多数Java程序员追逐像Singleton、Decorator、Observer这样的设计模式，而不重视面向对象的分析和设计。甚至还有经验丰富的Java程序员没有听说过OOPS和SOLID设计原则，他们根本不知道设计原则的好处，也不知道如何依照这些原则来进行编程。

众所周知，Java编程最基本的原则就是要追求高内聚和低耦合的解决方案和代码模块设计。查看Apache和Sun的开放源代码能帮助你发现其他 Java设计原则在这些代码中的实际运用。Java Development Kit则遵循以下模式：BorderFactory类中的工厂模式、Runtime类中的单件模式。你可以通过Joshua Bloch的《Effective Java》一书来了解更多信息。我个人偏向的另一种面向对象的设计模式是Kathy Sierra的Head First Design Pattern以及Head First Object Oriented Analysis and Design。

虽然实际案例是学习设计原则或模式的最佳途径，但通过本文的介绍，没有接触过这些原则或还在学习阶段的Java程序员也能够了解这10个面向对象的设计原则。其实每条原则都需要大量的篇幅才能讲清楚，但我会尽力做到言简意赅。

OrzEye for android， 利用手机摄像头取词并识别翻译

中文名“囧眼识词”，模仿的是有道词典的一个功能，记得之前腾讯有个QQ慧眼，也是相似的功能。

利用摄像头，对准某个单词，然后OCR，对识别结果进行处理，然后就是查词典了。

其实难点在于OCR的准确率和后期对识别结果的智能纠错。读研期间毕竟做了那么久的车牌字符识别，也算有点心得，所以就用这个创意拿来练手了。

搜索的时候发现了tess-two，是一个Tesseract的安卓版本。Tesseract这个开源软件读研时就听说过。

但是当时想多学点东西，不想直接拿现成的来用，所以那会OCR的训练和识别都是自己做的。

但是这次我试用了tess-two后发现这个东西相当不错，识别率比我那会做的东西好多了。

后悔了，早知道当时拿这个开源的东西来糊弄导师不就好了。
