OutOfMemoryError 实践与分析

继续抄书. 前文介绍了JVM运行时内存区的分配. 本文将介绍 OutOfMemoryError (OOM).

由前文可知, JVM运行时内存可分为:程序计数器、Java虚拟机栈、本地方法栈、Java堆、方法区(包含常量池). 除了程序计数器外，虚拟机内存的其他几个运行时区域都有发生OutOfMemoryError异常的可能. 本文通过实例验证各个区域OutOfMemoryError异常发生的场景. 文中的代码都是基于Sun Hotspot虚拟机1.6版的实现，对于不同公司的不同版本的虚拟机，参数与程序运行结果可能结果会有所差别。

1.JVM内存设置参数

-Xms :设置JVM初始化堆内存大小

-Xmx :设置JVM最大允许分配的堆内存大小

-Xss :设置JVM栈内存大小

-XX:PermSize : 设置JVM初始分配的非堆内存

-XX:MaxPermSize 设置:JVM最大允许分配的非堆内存

Java 运行时数据区域(续)

前一篇文章（Java 运行时数据区域）中介绍了 Java 运行时的各个数据区域,本文先将前文做简要总结,在以实际例子分析Java运行时的数据区域.

1.总结

  • 寄存器
    每一个线程都有一个独立的程序计数器,保存Java虚拟机正在执行的字节码指令的地址，若方法是native的，那程序计数器的值是undefined.
  • 栈
    每一个线程都有自己私有的Java虚拟机栈.栈帧(Stack Frame)用于存储局部变量表、操作栈、动态链接、方法出口等信息.局部变量表存放了编译期可知的各种基本数据类型、对象引用和字节码指令地址
  • 堆
    Java堆可由各条线程共享,存储类实例和数组对象.由Garbage Collector(垃圾收集器)管理这些对象
  • 静态域
    存放静态成员(static定义的).在方法区中
  • 常量池
    存放字符串常量和基本类型常量(public static final).在方法区中

Java volatile 变量的可见性与原子性浅析 

最近在看<Java Concurrency In Practice>, 对Java volatile变量有了新的认知下面综合各种资料浅显地总结一下我对volatile的理解。

##
1.Java内存模型

Java是运行于JVM之上的语言，有着自己的内存模型，这就可以将不同操作系统和不同硬件平台实现细节屏蔽掉，进而达成跨平台的目的。







在Java中，所有实例域、静态域和数组元素存储在堆内存中，堆内存在线程之间共享。局部变量、方法定义参数和异常处理器参数不会在线程之间共享，它们不会有内存可见性问题，也不受内存模型的影响。Java线程之间的通信由Java内存模型（JMM）控制Java内存模型的抽象示意图如下：







![](https://otwrpg.dm2302.livefilestore.com/y2pMX1mvZWXwsIxnT1a2s2C3kZGyRt8tKMS9R7LFJnuFIrSQBc5ZHKZEG3heRNDzeubNRgOFO7VwZZzSpwMkXcjUL9tZ_YeYzXx1Fs1NVT8F5s/11.png?psid=1)







Java内存模型规定，对于多个线程共享的变量，存储在主内存当中，每个线程都有自己独立的工作内存，线程只能访问自己的工作内存，不可以访问其它线程的工作内存。工作内存中保存了主内存共享变量的副本，线程要操作这些共享变量，只能通过操作工作内存中的副本来实现，操作完毕之后再同步回到主内存当中。

Java中 HashMap 的原理与简单实现 

基本来源于《Thinking in Java》，记下来也算是简单复习一遍。

##
1.HashMap原理的简单分析 

HashMap之所以有较快的查询速度，是因为HashMap利用了key的hashcode存储key-value。






HashMap的内部结构，可以简单描述为一个list数组（在简单版本的 HashMap 实现中，数组大小固定）。






![](https://otwrpg.dm2301.livefilestore.com/y2pqA-0VuzeLothAGW9tNloBgnuJhucbU4YrYsABSWmzy1C0hLWeYOk_BRYJYFUBC0NkswFyc_f9kpcC6lJEVec7mdb6M3qL3TXKleIEtUDXmM/100542_Kf3E_134062.jpg?psid=1)<!-- more -->






通过计算key的hashcode，进而生成一个数组下标。由于数组容量是固定的，所以不能避免不同的key-value计算出相同的数组下标。






但由于数组本身保存的并不是key-value本身，而是一个list，所以下标相同的key-value会被依次放入同一个list中。






查询的时候，需要对list进行线性查询，但如果散列函数较好，每个list就会有很少的key-value，所以即使是线性查询也不会花太多时间。

##
2.HashMap的简单实现 

下面基本是书上的东西，自己合上书敲了一遍，加深记忆。






import java.util.AbstractMap;
import java.util.HashSet;
import java.util.Iterator;
import java.util.LinkedList;
import java.util.Set;
import java.util.Map;

public class SlowHashMap<k, v> extends AbstractMap<k, v> {
    static final int SIZE = 100;
    @SuppressWarnings("unchecked")
    LinkedList<mapentry<k, v>>[] buckets = new LinkedList[SIZE];

    public V put(K key, V value) {
        V oldValue = null;
        if(key == null) {
            MapEntry<k, v=> nullElement = new MapEntry<k, v>(null, null);
            buckets[0].add(0, nullElement);
            return oldValue;
        }
        int index = Math.abs(key.hashCode()) % SIZE;
        if(buckets[index] == null) {
            buckets[index] = new LinkedList<mapentry<k, v>>();
        }

        MapEntry<k, v> element = new MapEntry<k, v>(key, value);
        Iterator<mapentry<k, v>>  it = buckets[index].listIterator();
        boolean found = false;

        while (it.hasNext()) {
            MapEntry<k, v> temp = it.next();
            if(temp.getKey().equals(key)) {
                oldValue = temp.getValue();
                temp.setValue(value);
                found = true;
                break;
            }
        }

        if(!found) {
            buckets[index].add(element);
        }

        return oldValue;
    }

    public V get(Object key) {
        int index = Math.abs(key.hashCode()) % SIZE;
        if(buckets[index] == null) {
            return null;
        }
        for(MapEntry<k, v> e : buckets[index]) {
            if(e.getKey().equals(key)) {
                return e.getValue();
            }
        }
        return null;

    }
    @Override
    public Set<java.util.map.entry<k, v>> entrySet() {
        Set<java.util.map.entry<k, v>> set = new HashSet<map.entry<k,v>>;();
        for(LinkedList<mapentry<k, v>> bucket : buckets) {
            if(bucket == null) {
                continue;
            }
            for(MapEntry<k, v> pair : bucket) {
                set.add(pair);
            }
        }

        return set;
    }


    @SuppressWarnings("hiding")
    class MapEntry<k, v> implements Map.Entry<k, v>{
        private K key;
        private V value;


        public MapEntry(K key, V value) {
            this.key = key;
            this.value = value;
        }
        @Override
        public K getKey() {
            return key;
        }

        @Override
        public V getValue() {
            return value;
        }

        @Override
        public V setValue(V value) {
            V result = this.value;
            this.value = value;
            return result;
        }

    }

}
