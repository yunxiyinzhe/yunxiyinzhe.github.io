Java 内部类 基础知识总结

将一个类定义在另一个类的内部，即是 内部类 。内部类的用途主要有：

  • 内部类能够提供更好的封装，将内部类隐藏在外部类之内。
  • 内部类可以访问外部类的所有成员数据，包括私有数据。
  • 匿名内部类适合于创建那些只需要使用一次的类。

根据内部类定义位置的不同，可以将内部类分为一下几类：

  • 普通内部类：定义在外部类中的非静态内部类
  • 静态内部类：定义在外部类中的静态内部类，又叫做嵌套类
  • 局部内部类：在方法中定义的内部类
  • 匿名内部类：不给出类名，直接定义一个类，通常这个类实现了某种接口或者抽象

下面将对以上几种内部类分别做简要的总结。

PS：发现自己Java基础很不牢固啊，在写博文的时候，发现自己对Java类的修饰符和方法修饰符的概念有些混淆了，这里再补一下基础吧。
Java的权限修饰赋：

>
>

> public：可以修饰类、接口、变量、方法。public类必须定义在和类名相同的同名文件中,即一个Java文件中只能有一个类可以被修饰为public。
>



> protected：可以修饰变量、方法，不能修饰外部类只能修饰内部类
>



> default：可以修饰类、接口、变量、方法
>



> private：可以修饰变量、方法，不能修饰外部类只能修饰内部类
>

因此除了内部类，普通类的访问权限修饰符只有public和默认权限。另外，在涉及继承关系的时候，子类重写父类的方法的访问修饰符，不能低于父类的方法访问权限。

普通内部类

下面的示例程序定义了一个简单的普通内部类，普通内部类可以看作为与外部类的(非静态)变量、(非静态)方法和(非静态)初始化块相似的类成员。因此与外部类只能使用public和默认权限两种访问权限修饰符不同，内部类可以使用任意访问权限修饰符。

class Outer{
    public class Inner1 {}
    private class Inner2 {}
    protected class Inner3 {}
    class Inner4 {}
}

如何创建内部类的对象呢，看下面的代码

class Outer {
    public class Inner {}
    Inner createInner() {
    Inner i = new Inner();
    return i;
    }
    static Inner createInnerStatic() {
    Outer.Inner i = (new Outer()).new Inner();
    return i;
    }
}
class Outer2 {
    Outer.Inner createInner() {
    Outer.Inner i = (new Outer()).new Inner();
    return i;
    }
}

如果是在内部类所在的外部类中的非静态域中创建内部类对象，和普通类的创建方法没有什么不同。如果在静态域或者其他类中创建内部类对象，首先要指明这个对象的具体类型：Outer.Inner，此外还要使用外部类对象来创建内部类对象，就像上面程序所展示的那样。

为什么内部类对象如此依赖于外部类呢？我们知道，内部类对象自动拥有对其外部类所有成员的访问权限。为了实现这个目的，内部类必须持有创建他的那个外部类的对象的引用，内部类就是通过这个引用来访问外部类的成员变量的。因此内部类对象只能在与外部类的对象相关联的情况下才会被创建。在外部类的非静态域创建内部类对象时，外部类会隐式地使用Outer.this创建内部类，在静态域中则必须显式地使用”.new“语法。看下面的代码

class Outer {
    String s = "Outer s";
    String t = "Outer t";
    public class Inner {
    String s = "Inner s";
    public Outer getOuter() {
        return Outer.this;
    }
    public void print() {
        System.out.println(s);//Inner s
        System.out.println(Outer.this.s);//Outer s
        System.out.println(t);//Outer t
    }
    }
    public static void main(String[] args) {
    Outer o = new Outer();
    Outer.Inner i = o.new Inner();
    System.out.println(o == i.getOuter());
    i.print();
    }
}

输出是true，进一步证实了外部类会将自身的引用传入内部类中。此外在内部类中this指的是内部类对象的this，如果想得到外部类对象的引用，则需要使用Outer.this。在内部类中使用外部类的变量，如果有同名变量，则要显式地使用Outer.this指出，其他情况可以省略。

也可以将接口定义在类的内部，并且可以使用private权限修饰符，以获得更好的封装特性。

class Outer {
    private interface Inner {
    void in();
    }
    public class InnerImp implements Inner {
    @Override
    public void in() {
        System.out.println("InnerImp");
    }
    }
}

静态内部类

如果使用static来修饰一个内部类，则这个内部类对象就与外部类对象没有联系了，通常称作静态内部类或者嵌套类。普通的内部类隐式地保存了指向创建他的外部类对象的引用，而静态内部类则没有。静态内部类可以包含静态或非静态的成员，但是不能访问外部类的实例成员，只能访问其类成员。即使是静态内部类的实例方法也只能访问外部类的静态成员。此外，普通内部类不能有static数据和static方法，也不能有静态内部类，而静态内部类可以包含所有这些东西。

class Outer {
    public static class InnerStatic {
    int i;
    static int j;
    public void test() {}
    public static void testStatic() {}
    static class InnerInnerStatic {}
    class InnerInner {}
    }
    public static void main(String[] args) {
    new InnerStatic().i = 1;
    new InnerStatic().test();
    InnerStatic.j = 1;
    InnerStatic.testStatic();
    }
}

Java允许在接口中定义内部类，接口中定义的内部类默认是public和static的。如果想要创建某些公共代码，使得他们可以被某个接口的所有不同实现所共用，那么使用接口内部的嵌套类就会显得方便。

interface Outer {
    void out();
    class Inner implements Outer {
    @Override
    public void out() {
        System.out.println("outer implement");
    }    
    public void in() {
        System.out.println("in method");
    }    
    }
}

局部内部类

如果把一个内部类放在方法里面定义，则这个内部类就是一个局部内部类。局部内部类只在该方法内有效。由于局部内部类不能在外部类的方法以外的地方使用，因此局部内部类不能使用权限修饰符和static修饰符。不管是局部变量还是局部内部类，他们的上一级程序单元都是方法而不是类，所以用static修饰他们没有意义。局部变量或局部内部类的作用域是其所在方法，不能使用任何权限控制符。

class Outer {
    public static void main(String[] args) {
    class Inner {
        int a = 0;
    }
    Inner i = new Inner();
    System.out.println(i.a + 1);
    }
}

匿名内部类

匿名内部类适合创建那种只需要一次使用的类，创建匿名内部类时会立即创建一个该类的实例，这个类的定义立即消失，因此匿名内部类不能重复使用。匿名内部类可以继承一个父类或者实现一个接口，但最多只能继承一个父类或实现一个接口。

class Outer {
    void test1 (TestInterface inn) {
    inn.f();
    }
    void test2 (TestClass inn) {
    inn.f();
    }
    public static void main(String[] args) {
    Outer o = new Outer();
    o.test1(new TestInterface() {
        @Override
        public void f() {
               System.out.println("Implement interface");
        }
    });

    o.test2(new TestClass() {
        @Override
        public void f() {
           System.out.println("Implement interface");
        }
    });
    }
}
interface TestInterface {
    void f();
}
class TestClass {
    void f() {};
}

当通过实现接口来创建匿名内部类时，匿名内部类不能显示创建构造器，因此匿名内部类只有一个隐式的无参数构造器，故new接口名后面的括号内不能传入参数值。当通过继承父类来创建匿名内部类时，匿名内部类拥有和父类相似的构造器，相似是指拥有相同的形参列表。

class Outer {
    void test2 (TestClass inn) {
    inn.f();
    }
    public static void main(String[] args) {
    Outer o = new Outer();
    o.test2(new TestClass() {
        @Override
        public void f() {
            System.out.println(s);
        }
    });//123
    o.test2(new TestClass("456") {
        @Override
        public void f() {
            System.out.println(s);
        }
    });//456
    }
}
class TestClass {
    String s = "123";
    public TestClass() {}
    public TestClass(String s) {
    this.s = s;
    }
    void f() {};
}

如果匿名内部类要访问外部类的局部变量，则必须使用final修饰符来修饰外部局部变量，否则编译器会报错。

内部类的继承

因为内部类的构造器必须连接到指向外部类对象的引用，所以在继承内部类的时候，必须使用特殊的语法来说明清出内部类外部类和导出类之间的关系

class Outer {
    class Inner {}
}
class SubInner extends Outer.Inner {
    //默认构造器和无参数构造器都会编译错误
    public SubInner(Outer o) {
    o.super();
    }
    public static void main(String[] args) {
    Outer o = new Outer();
    SubInner si = new SubInner(o);
    }
}

内部类的覆盖

通过下面的例子可以看到内部类被覆盖后会发生什么事情。

class Outer {
    class Inner {
    void test() {
        System.out.println("Inner");
    }
    }
}
public class SubOuter extends Outer {
    class SubInner extends Outer.Inner {
    @Override
    void test() {
        System.out.println("SubInner");
        }
    }
    public static void main(String[] args) {
    SubOuter.Inner i = (new SubOuter()).new SubInner();
    i.test();
    }
}

运行输出结果为“SubInner”，SubInner的test方法覆盖了父类的方法。
